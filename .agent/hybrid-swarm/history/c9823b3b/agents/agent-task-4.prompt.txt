# Task: City system with population, tax, warehouse, and upgrades

Implement src/world/city.rs: City struct with fields: id (CityId), position (Vec2), radius (~30px), population (f32, range 50-500), tax_rate (f32, 0.0-0.15), treasury (f32, accumulated tax), warehouse (HashMap<Commodity, f32>, capacity 500 total), prosperity (f32, 0-100), specialization (one random Commodity category for 1.5× crafting speed), upgrades (HashSet of MarketHall, Walls, Harbor, Workshop), ticks_without_food (u32), is_coastal (bool). Methods: (1) tick_population — grow by 0.01/tick when prosperity > 60 and food stocked (GRAIN, FISH, PROVISIONS > 0), decline by 0.02/tick when no food for 200+ ticks. (2) tick_warehouse — decay overstocked goods by 0.1%/tick. (3) tick_tax_adjustment — every 500 ticks: raise tax by 1% if trade volume above avg, lower by 1% if below, clamp to [0, 0.15]. (4) compute_prosperity — from population, trade volume, warehouse fullness, commodity diversity. (5) try_purchase_upgrade — deduct from treasury if affordable, apply effects. (6) City placement via Poisson disk sampling (static method generating N cities with minimum spacing). Coastal cities determined by terrain.

Target files: src/world/city.rs

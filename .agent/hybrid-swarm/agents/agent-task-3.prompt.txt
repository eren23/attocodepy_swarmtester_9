# Task: Integration tests for merchant gray-tile avoidance

Write integration tests verifying merchants no longer get stuck on gray (impassable) tiles. Create a new test file tests/test_pathfinding.rs with:

1. **test_pathfinding_around_mountain_range** — Construct a small terrain grid with a mountain range separating two passable areas. Call terrain.find_path() and assert a path is returned that avoids all impassable cells.

2. **test_unreachable_destination_returns_none** — Create terrain where goal is completely surrounded by mountains. Assert find_path() returns None and is_reachable() returns false.

3. **test_merchant_does_not_stick_to_gray_tiles** — Set up a minimal world with 2 cities separated by a mountain range. Spawn a merchant at city A targeting city B. Run 500+ ticks and assert: (a) merchant is never on an impassable tile, (b) merchant eventually reaches city B or is making progress (position changes over any 50-tick window). Use the existing test helpers in tests/common/ for world setup.

4. **test_path_prefers_faster_terrain** — Create terrain with a short path through hills (slow) and a longer path through plains (fast). Assert the path through plains is chosen when the cost difference favors it.

Target files: tests/test_pathfinding.rs
